# Joom 프로젝트 아키텍처 및 세부 정보

## 1. 프로젝트 개요

Joom은 C++17과 SDL2 라이브러리를 사용하여 만든 둠(Doom) 스타일의 1인칭 3D 게임 엔진입니다. 레이캐스팅 기술을 사용하여 3D 환경을 렌더링하며, 기본적인 게임플레이 메커니즘을 갖추고 있습니다.

## 2. 핵심 아키텍처

프로젝트는 중앙 집중식 아키텍처를 따릅니다. `Game` 클래스가 모든 주요 시스템을 소유하고 관리하는 중심 허브 역할을 합니다.

- **`Game` 클래스**: 엔진의 메인 클래스입니다. 게임 루프를 실행하고, 모든 하위 시스템(렌더러, 플레이어, 맵 등)을 초기화하며, 시스템 간의 데이터 흐름을 조정합니다.
- **게임 루프**: `handleEvents()`, `update()`, `render()`의 세 단계로 구성된 표준적인 게임 루프를 따릅니다.
    1.  **`handleEvents()`**: 플레이어 입력(키보드, 마우스) 및 창 이벤트를 처리합니다.
    2.  **`update()`**: 플레이어 위치, 몬스터 AI, 아이템 상태 등 게임 월드의 상태를 갱신합니다.
    3.  **`render()`**: 현재 게임 상태를 화면에 그립니다.

## 3. 주요 컴포넌트 (클래스)

각 컴포넌트는 특정 역할을 수행하며 `Game` 클래스에 의해 관리됩니다.

| 클래스 | 헤더 파일 | 소스 파일 | 역할 및 책임 |
| --- | --- | --- | --- |
| **`Game`** | `Game.h` | `Game.cpp` | 모든 시스템을 총괄하는 중앙 클래스. 게임 루프 실행. |
| **`Renderer`** | `Renderer.h` | `Renderer.cpp` | 2D 맵과 레이캐스팅을 사용해 3D 월드를 렌더링. 바닥, 천장, 벽, 스프라이트(아이템, 몬스터)를 그림. |
| **`Player`** | `Player.h` | `Player.cpp` | 플레이어의 위치, 방향, 이동, 충돌 감지를 관리. |
| **`Map`** | `Map.h` | `Map.cpp` | 게임 맵 데이터를 로드하고 관리. 벽, 아이템, 몬스터의 위치 정보를 저장. |
| **`AudioManager`**| `AudioManager.h`| `AudioManager.cpp`| 배경 음악 및 사운드 이펙트(발소리, 문 소리 등)를 재생하고 관리. |
| **`Monster`** | `Monster.h` | `Monster.cpp` | 몬스터의 상태(위치, 체력)와 행동을 관리. |
| **`Pathfinder`**| `Pathfinder.h` | `Pathfinder.cpp` | A* 알고리즘을 사용하여 몬스터가 플레이어를 찾아가는 경로를 계산. |
| **`HUD`** | `HUD.h` | `HUD.cpp` | Heads-Up Display. 플레이어의 체력, 무기, 미니맵 등을 화면에 표시. |
| **`ItemManager`**| `ItemManager.h`| `ItemManager.cpp` | 게임 내 아이템(열쇠, 무기 등)의 상태를 관리하고 상호작용을 처리. |
| **`LightSystem`**| `LightSystem.h`| `LightSystem.cpp` | 게임 내 조명 효과(예: 손전등)를 관리. |
| **`TextureManager`**|`TextureManager.h`|`TextureManager.cpp`| 벽, 아이템 등의 텍스처를 로드하고 관리하여 메모리를 효율적으로 사용. |

## 4. 빌드 시스템

- **CMake**: `CMakeLists.txt` 파일을 사용하여 프로젝트를 빌드합니다.
- **의존성**:
    - `SDL2`: 창 생성, 이벤트 처리, 렌더링 등 기본적인 그래픽 처리를 위해 사용됩니다.
    - `SDL2_image`: PNG와 같은 이미지 파일을 로드하기 위해 사용됩니다.
    - `SDL2_mixer`: MP3, WAV 형식의 오디오 파일을 재생하기 위해 사용됩니다.

## 5. 주요 디렉토리 구조

- **`/src`**: 모든 `.cpp` 소스 파일이 위치합니다.
- **`/include`**: 모든 `.h` 헤더 파일이 위치합니다.
- **`/textures`**: 게임에 사용되는 텍스처 이미지 파일 (PNG)이 저장됩니다.
- **`/sounds`**: 배경 음악 및 효과음 파일 (WAV, MP3)이 저장됩니다.
- **`/build`**: 빌드 결과물(실행 파일 등)이 생성되는 디렉토리입니다.

이 파일은 프로젝트의 현재 상태를 기준으로 작성되었으며, 기능이 추가되거나 변경되면 업데이트될 수 있습니다.
